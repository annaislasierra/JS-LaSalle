Forma correcta de treballar amb GitHub (jo com a propietària de projecte)

1. Crear el repo a GitHub

  1. Botó verd New.
  2. Opcions: millor Public (total...).
  3. Sempre amb el README file marcat.
  4. No calen .gitignore ni license.
  5. Botó verd Create repository.

2. Clonar el repositori

  1. Copiar l'enllaç del repo en remot (a partir del botó verd Code).
  2. Obrir la carpeta en local a VSC.
  3. A l'arxiu Welcome de VSC, marcar l'opció Clone Git Repository.
  4. Al camp de text duperior central, enganxar l'URL del repo y Enter.
  5. Indicar la carpeta local on es clonarà el repo.

Es pot usar l'arxiu README.md per guardar un petit índex del contingut, per exemple, usant el llenguatge de marques MarkDown:

  • ## -> Per a títols (estil h1 o h2).
  • Un guió amb un espai -> Crea llistes amb punts.
  • [Texte](adreça https) -> Crea enllaços a URL. Apareix el "Texte" en un altre color; clic i envia a la URL entre parèntesi.

Després de fer canvis en local, icona Source Control de VSC, afegir un missatge descriptiu i Commit; sincronitzar i dir que sí (per pujar al remot els canvis).

SEMPRE afegir un missatge al commit; si no, es penja (al menys el meu).

Dreceres de teclat molt molones

  • ! Tab (o Enter): en un arxiu en blanc, genera l'estructura bàsica d'una pàgina html. Compte! La funcionalitat Emmet ha d'estar habilitada (ho està per defecte) y l'arxiu ha de tenir l'extensió html.
  • log + 1.ª opció: escriu automàticament console.log().
  • F12 o Majúsc Ctrol I: obrir la consola del navegador.

Extensions útils

  • Open in Default Browser: apareix aquesta opció en fer clic dret sobre una pàgina. Si es fan canvis, cal recarregar-la. Una altra opció es Live Server.
  • Live Server: Inicia un servidor en local pitjant la iconeta Go Live a la línia inferior dreta de VSC. Actualitza canvis automàticament.
